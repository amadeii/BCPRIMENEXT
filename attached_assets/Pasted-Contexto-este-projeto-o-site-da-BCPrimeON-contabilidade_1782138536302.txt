## 2. Planos: usar a estrutura do Plano de Negócio (Central Starter/Growth/Scale/Enterprise)

Substitua os 3 planos atuais (Essencial R$99 / Profissional R$199 / Empresarial R$399) pelos 4 planos abaixo, definidos no Plano de Negócio Definitivo:

- **Central Starter** — R$ 195/mês — "Ideal para MEI"
  Features: Abertura de empresa grátis (com 12 meses de assinatura); Contabilidade completa; ERP básico (1 usuário, estoque, financeiro); Emissão de até 500 notas fiscais/mês; Consultoria fiscal com Maestro IA 24/7; Suporte via WhatsApp; Relatórios gerenciais básicos.

- **Central Growth** — R$ 395/mês — "Para empresas do Simples Nacional em crescimento" (marcar como `popular`)
  Features: Tudo do Central Starter; ERP completo (5 usuários, estoque avançado, financeiro, vendas); Emissão de até 1.000 notas fiscais/mês; Consultoria de crescimento mensal; Consultoria de transição de regime tributário; Suporte prioritário via WhatsApp; Relatórios gerenciais avançados; Maestro IA com análise de cenários.

- **Central Scale** — R$ 795/mês — "Para empresas do Lucro Presumido"
  Features: Tudo do Central Growth; ERP enterprise (10 usuários, estoque enterprise, financeiro avançado); Emissão de até 5.000 notas fiscais/mês; Consultoria estratégica semanal; Consultoria de transição para Lucro Real; Suporte dedicado (telefone + WhatsApp); BI e análise avançada; Maestro IA com previsões e automações.

- **Central Enterprise** — "Sob consulta" — "Para empresas do Lucro Real"
  Features: Tudo do Central Scale; ERP completo ilimitado; Consultoria estratégica dedicada; Integração com sistemas externos; BI customizado; Maestro IA com automações avançadas; Suporte 24/7.

Mantenha a seção de FAQ existente na página de planos, mas remova ou ajuste a frase "Os planos apresentados são exclusivos para empresas optantes pelo Simples Nacional" já que agora há um plano para Lucro Presumido (Scale) e um para Lucro Real (Enterprise sob consulta).

## 3. Painel de gerenciamento de planos (para vocês ajustarem sem precisar tocar no código)

Hoje os planos estão hardcoded como um array dentro de `client/src/pages/PlanosPrecos.tsx`. Construa um CRUD completo, no mesmo padrão usado para `blogPosts` e `teamMembers`:

a) **Schema** (`shared/schema.ts`): criar uma tabela `plans` com colunas: id (varchar, pk, default gen_random_uuid()), name (text), slug (text, unique), description (text), price (text — para suportar "195" e também "Sob consulta"), billing (text, ex: "/mês"), popular (boolean, default false), features (text — salvar como JSON stringificado de `{ name: string; included: boolean }[]`), ctaLabel (text), ctaHref (text), displayOrder (integer ou text, default "0"), active (boolean, default true), createdAt/updatedAt (timestamp). Gerar `insertPlanSchema` com createInsertSchema (omitindo id/createdAt/updatedAt) e exportar os tipos `Plan`/`InsertPlan`, seguindo exatamente o padrão de `teamMembers`/`insertTeamMemberSchema` já existente no arquivo.

b) **Storage** (`server/storage.ts`): adicionar ao `IStorage` e à implementação: `getPlans()` (todos, ordenados por displayOrder), `getActivePlans()` (apenas active=true, para a página pública), `getPlan(id)`, `createPlan(plan)`, `updatePlan(id, plan)`, `deletePlan(id)`. Incluir `plans` na contagem retornada por `getStats()`.

c) **Rotas** (`server/routes.ts`):
   - `GET /api/plans` → público, retorna `storage.getActivePlans()`, usado pela página `/planos-e-precos`.
   - `GET /api/admin/plans` → todos os planos (inclusive inativos), para a listagem no admin.
   - `GET /api/admin/plans/:id` → um plano.
   - `POST /api/admin/plans` → cria, validando com `insertPlanSchema.safeParse`.
   - `PATCH /api/admin/plans/:id` → atualiza.
   - `DELETE /api/admin/plans/:id` → remove.
   Seguir exatamente o padrão de erro/validação já usado nas rotas de `/api/admin/team`.

d) **Admin UI**: criar `client/src/pages/admin/AdminPlans.tsx` (listagem em tabela, com nome, preço, status ativo/inativo, botões de editar/excluir — mesmo padrão visual de `AdminPosts.tsx`, incluindo o `AlertDialog` de confirmação de exclusão) e `client/src/pages/admin/AdminPlanForm.tsx` (formulário de criar/editar com campos para nome, slug, descrição, preço, billing, popular (switch), active (switch), CTA label/href, e uma lista editável de features onde cada item tem um texto e um toggle "incluído" — permitir adicionar/remover linhas de feature dinamicamente). Registrar as rotas `/admin/plans` e `/admin/plans/novo` e `/admin/plans/:id/editar` em `App.tsx`, e adicionar um card "Planos" no `AdminDashboard.tsx` (mesmo padrão dos cards de Posts/Keywords/Leads/Equipe), usando o ícone `Tag` ou `DollarSign` do lucide-react.

e) **Página pública**: refatorar `client/src/pages/PlanosPrecos.tsx` para buscar os planos via `useQuery<Plan[]>({ queryKey: ["/api/plans"] })` em vez do array hardcoded hoje existente, ordenando por displayOrder, fazendo `JSON.parse` do campo `features`, e mantendo o layout de cards atual (incluindo o destaque visual quando `popular === true`).

f) **Seed**: atualizar `server/seed.ts` para, se a tabela `plans` estiver vazia, inserir os 4 planos da seção 2 acima (Central Starter, Growth, Scale, Enterprise) já com `displayOrder` 1-4 e `popular: true` no Growth.

g) Depois de criar o schema, rodar `npm run db:push` para aplicar a migração no banco antes de testar.
